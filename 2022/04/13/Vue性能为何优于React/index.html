Vue性能为何优于React
2022-04-13 22:37:02 # 工程化

引言

​ 上述组件被挂载时会触发 this.msg='hello world' ，此时会触发组件的更新，视图重新渲染，但是父组件中状态的更新，但是作为子组件的 <ChildComponent /> 其实不会重新渲染，状态更新，组件不会重新渲染，这是 Vue 性能比 React 高的根本原因。

React 的更新粒度

​ 当不手动进行性能优化时，React 在父组件状态更新时，会自上而下递归式的进行更新，换句话说，如果 <ChildComponent /> 这个组件中嵌套有多个组件，这些组件下层又嵌套有多个组件，每层都会重新 render ，这无疑是毁灭性的性能灾难。

​ 对此 React 引入了纯组件等要求开发者手动进行性能优化的方案，在 16.8 版本中引入了 fiber ，创造了异步渲染，但本质上都是在填当初设计思想层面埋下的隐患。

Vue 的更新粒度

​ Vue 在设计思想层面并没有采用 React 的纯 diff 方案，而是为每个组件创造了自己的渲染 watcher，通过这个 watcher ，来控制视图的更新，对应前文的场景，父组件的 watcher 不会掌管子组件的更新。

Immutable 双刃剑

​ React 为什么不借鉴 Vue 的依赖收集思想，从而减轻开发者心智负担，从设计层面优化性能呢。因为 React 遵从 Immutable 思想，即所有数据都是不可变的，如果需要修改数据则需要拷贝这一份数据，可变数据类型能够节省内存或是利用可变性做一些有用的事情，但是在函数式编程中，这样做的好处远远大于其副作用，React 又是采用 F(state) = UI 这一思想，所以 Immutable 是 React 不可抛弃的思想。

Immutable Data 是指一旦被创造后，就不可以被改变的数据。

通过使用 Immutable Data，可以让我们更容易的去处理缓存、回退、数据变化检测等问题，简化我们的开发。

js 中的 Immutable Data

在 javascript 中我们可以通过 deep clone 来模拟 Immutable Data，就是每次对数据进行操作，新对数据进行 deep clone 出一个新数据。
